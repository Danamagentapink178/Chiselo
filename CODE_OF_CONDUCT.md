# Code of Conduct

Chiselo welcomes practical, respectful collaboration.

## Expected Behavior

- Be kind and specific.
- Critique code and design decisions, not people.
- Assume contributors may have different levels of programming experience.
- Keep issues focused on reproducible behavior, product direction, or concrete improvements.
- Respect the non-commercial license.

## Unacceptable Behavior

- Harassment, insults, threats, or personal attacks.
- Posting private files, credentials, or proprietary code.
- Pressuring maintainers for commercial support or commercial permission.
- Using issues or pull requests to advertise unrelated services.

## Enforcement

Maintainers may edit, hide, lock, or delete comments and issues that violate this code of conduct. Repeat or severe violations may lead to blocking from the project.
